# 申し送り書 個店マクロ改修（店型2の扱い統一・HB食品用調整・履歴保存先固定）

作成日: 2026-07-12

要件定義フェーズで依頼者と確定した要件の記録です。仕様の正本は
`要件定義書_在庫自動調整マクロ.md` / `設計書_在庫自動調整マクロ.md` であり、
本改修の反映後はそちらを参照してください。

## 1. 背景

当初依頼は以下の3点だった。

1. 店型２陳列数(本部推奨)の行の数量は、その半分を計算式に反映させる。
2. 店型２陳列数を反映しない（フェイス陳列数(本部推奨)のみを使用する）マクロ「HB（改行）食品用」を新設する。
3. AM維持フラグ履歴ブックの保存先も Downloads に固定する。同名ファイルがあれば追記していく現行仕様は維持する。

要件定義の質疑で、1. は依頼者の認識齟齬（店型2は加算だけに使われていると認識していたが、
実際は自動計算の対象判定にも合算で使用されていた）と判明し、**「半分反映」は廃止**。
正しい要件は「本部推奨の参照をフェイス1のみに統一し、店型2は最終加算専用とする」に確定した。

## 2. 確定要件

### R1. 本部推奨参照のフェイス1統一（店型2は加算専用）

- 自動計算の対象判定：現行「本部推奨フェイス合計（フェイス1＋店型2）＞ 運用側下限値」→ **本部推奨フェイス1のみ ＞ 運用側下限値**
- 下限側の寄せ先・本部推奨優先の戻し（本部推奨が下限値より小さいときの処理）も**フェイス1のみ**で統一
- 店型2の加算は現行維持：**数値が入っていればそのまま加算（半分にしない）**
- 販売予定なし判定（フェイス1と店型2の両方0で週スキップ）は**現行維持**（通常の自動調整・手動調整）
- 「フェイス1= 0 かつ 店型2 > 0」の週（通常モード）は、計算・店型2加算を行わず、**シート上下限から外れた既存値の補正だけ**行う（2026-07-12 のレビュー質疑で確定。改修前はこの週も店型2の値で計算対象だった）
- 最終書込値のシート上下限クランプは**全経路で必ず維持**（本部下限値を下回る値は社内システムにはじかれるため）

**影響範囲（依頼者確認済みの挙動変化）**：フェイス1 ≤ 下限値 ＜ 合計 のSKUは、
従来「通常の自動計算対象」だったが、今後は「本部推奨優先の戻し」経由になる。
この判定は販売計画の自動計算も連動して抑止する（フェイス1 ≤ 下限値のSKUは販売計画を書き換えない）。
統一しない場合「フェイス1 ≤ 下限値 ＜ 合計」の商品がどの処理にも拾われない穴が生じるため、統一が正。

### R2. 新マクロ「HB（改行）食品用」

- **手動調整ベース**：手持週数・最下限値を入力してから実行。対象列はZ列のみ。AM維持フラグ退避も既定実行（共通エンジン経由）
- **店型2を完全無視**：加算しない ＋ 販売予定なし判定もフェイス1のみ（フェイス1が0の週は、店型2に数値があってもスキップ）
- リボンボタン label は改行入りの `HB&#xA;食品用`
- 調整後ブックのファイル名は手動調整と**同形式・サフィックスなし**
  （`mmdd商品名4文字入力名_X.X週_下限値N[_未意思入のみ]`）。同名既存ファイルは削除して保存する現行仕様のまま
- ダイアログ・完了メッセージ用のモード名は「HB食品用調整」とする

### R3. AM維持フラグ履歴ブックの保存先 Downloads 固定

- `ResolveKeepFaceHistoryFolder` を調整後ブックと同方式に変更：`GetDefaultDownloadFolder()`
  （`shell:Downloads` 最優先）固定。Downloads が見つからないPCのみフォルダ選択ダイアログへフォールバック
- 前回保存先の再利用（setting シート）と「この保存先を使いますか？」確認ダイアログは**廃止**
- **同名ファイル（`mmddAM維持フラグ商品履歴.xlsx`）があればそこへ追記・重複整理する現行仕様は変更しない**
- ダイアログのキャンセル時は現行どおり退避中止→調整全体中止

## 3. 実装時の要件外決定事項

| 項目 | 決定 |
| --- | --- |
| 店型2半分時の端数処理 | 半分反映の廃止により不要（質疑では切り上げを選択していた） |
| HB食品用のファイル名接尾辞 | 付けない（依頼者指定） |
| 本部推奨参照の関数 | `HqFaceTotal` は実態に合わせフェイス1のみ参照の `HqFaceBase` へ改名 |
| settingシート（modSettings） | 未使用化するがモジュールは汎用基盤として残す |

## 4. 動作確認チェックリスト（依頼者実施）

- 手動/自動調整：店型2入力SKUで加算値がそのまま乗る（半分にならない）
- フェイス1 ≤ 下限値 ＜ フェイス1＋店型2 のSKU（フェイス1 > 0）が「本部推奨優先の戻し」経由で処理される（従来との差分ポイント）
- フェイス1 = 0・店型2 > 0 の週（通常モード）は計算・加算されず、シート上下限から外れた既存値だけが補正される
- HB食品用：週数・下限値入力→実行で店型2が一切加算されない。フェイス1=0の週（店型2のみ入力）はスキップされる
- HB食品用の保存ファイル名が手動調整と同形式で、同名既存ファイルは置き換えられる
- AM維持フラグ退避：保存先ダイアログが出ずに Downloads へ保存。Downloads に同名の履歴ブックがあれば追記・重複整理される
- 全パターンで書込値がシート上下限内に収まる
