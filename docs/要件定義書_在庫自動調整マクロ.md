# 要件定義書 在庫自動調整マクロ

作成日: 2026-05-07
更新日: 2026-07-12

対象: `src/src.xlsm`、配布用 `releases/在庫自動調整.xlam`、`src/vba/mod*.bas`、および `src/ribbon/customUI14.xml`

## 1. 目的

店舗で使用する在庫調整用 Excel ブックに対して、販売計画とフェイス陳列数を一定ルールで補正し、MDシステムへ取り込める状態の調整後ブックを作成する。

特に、フェイス陳列数が本部推奨値より過度に下げられている場合は、指定下限値の大小だけで対象外にせず、本部推奨値まで戻す。

## 2. 対象範囲

対象機能は以下とする。

- 在庫自動調整
- 在庫手動調整
- 在庫HB食品用調整（店型２陳列数(本部推奨)を使用しない手動調整）
- 列単位脱色
- 局所脱色
- AM列フェイス維持フラグ退避（自動調整・手動調整・HB食品用調整の前処理として実行）
- 選択範囲のA4片面PDF出力
- 調整後ブックの名前変更保存
- エラー報告とデバッグログ出力

## 3. 前提条件

- 操作者が調整したいブックを Excel の前面に表示した状態で実行する。
- 対象ブックには `list` シートが存在する。
- 商品は `list` シート上で 22 行単位のブロックとして並ぶ。
- 商品ブロックの開始行は 8 行目とする。
- 自動調整の対象列は、2週間分の販売計画とフェイス陳列数として `Z` 列および `AA` 列とする。
- 手動調整・HB食品用調整の対象列は現在 `Z` 列のみとする。
- 脱色可能列は `Z` 列から `AK` 列までとする。
- フェイス陳列数は整数値として扱う。
- 数式セルはマクロによる値書き換え対象外とする。

## 4. 主要な行定義

販売計画行を `planRow` とした場合、主な参照行は以下とする。

| 用途 | 行位置 |
| --- | --- |
| 本部計画 | `planRow - 2` |
| 販売計画上限 | `planRow + 3` |
| 販売計画下限 | `planRow + 4` |
| フェイス陳列数上限 | `planRow + 6` |
| フェイス陳列数下限 | `planRow + 7` |
| 本部推奨フェイス1 | `planRow + 12` |
| 本部推奨フェイス2（店型2陳列数(本部推奨)） | `planRow + 13` |
| フェイス陳列数 | `planRow + 14` |

調整対象の判定と下限側の寄せ先に使う「本部推奨フェイス」は、本部推奨フェイス1のみとする。本部推奨フェイス2（店型２陳列数(本部推奨)）は、フェイス陳列数への最終加算専用とし、判定・寄せ先には使用しない。

シート上の「店型２陳列数(本部推奨)」は本部推奨フェイス2の行であり、マクロが入力するフェイス陳列数行の1行上に配置されている。例えば、フェイス陳列数行が 22 行目の場合、店型２陳列数(本部推奨)は 21 行目にある。

本書で「フェイス陳列数上限」と呼ぶ値は、`list` シート上の `planRow + 6` 行にあるフェイス陳列数の上限値を指す。空白または 0 の場合は、システム上限値 `300` をフェイス陳列数上限として扱う。

本書で「フェイス陳列数下限」と呼ぶ値は、`list` シート上の `planRow + 7` 行にあるフェイス陳列数の下限値を指す。

本書で「運用側最下限」と呼ぶ値は、自動調整では標準値 `5`、手動調整では操作者が入力する最低フェイス数を指す。フェイス陳列数下限とは別の値である。

## 5. 機能要件

### 5.1 実行入口

リボンから以下の処理を実行できること。

- 自動調整
- 手動調整
- HB食品用調整
- 列単位脱色
- 局所脱色
- 選択範囲PDF出力

AM維持フラグ退避の単独ボタンは設けない。AM維持フラグ退避は、自動調整・手動調整・HB食品用調整の実行時に既定で行う。

### 5.2 自動調整

自動調整は以下の固定条件で実行する。

- 目標手持週数: `2.0`
- フェイス陳列数の運用側最下限: `5`
- 修正対象列: `Z` 列および `AA` 列の2週間分

実行時は、SKU ブロックの処理前に AM維持フラグ退避を既定で行う（5.15 参照）。

### 5.3 手動調整

手動調整では、操作者が以下を入力できること。

- 目標手持週数
- フェイス陳列数の運用側最下限

入力値が数値でない場合、または許容範囲外の場合は処理を中止する。

実行時は、自動調整と同様に SKU ブロックの処理前に AM維持フラグ退避を既定で行う（5.15 参照）。

### 5.4 HB食品用調整

HB食品用調整は、店型２陳列数(本部推奨)を一切使用せず、フェイス陳列数の計算に本部推奨フェイス1のみを使用する手動調整である。

- 操作者が目標手持週数とフェイス陳列数の運用側最下限を入力する（5.3 と同じ入力・検証）。
- 修正対象列は手動調整と同じ `Z` 列のみとする。
- 店型２陳列数(本部推奨)の加算（5.9 参照）は行わない。
- 販売予定なし判定は本部推奨フェイス1のみで行う。本部推奨フェイス1が 0 の対象週は、店型２陳列数(本部推奨)に数値があっても販売予定なしとして扱い、一切書き換えない。
- 上記以外の計算・修正範囲の選択・保存・AM維持フラグ退避（5.15 参照）は手動調整と同じとする。
- 調整後ブックの保存名は手動調整と同形式とし、専用の接尾辞は付けない。

### 5.5 修正範囲の選択

自動調整・手動調整・HB食品用調整のすべてで、意思入れ済み商品を含めるかを操作者が選択できること。

現在の実装では、Excel 標準 `MsgBox` の制約によりボタン自体の文字は `はい` / `いいえ` / `キャンセル` となる。画面表示上の意味は以下とする。

| ボタン | 表示上の意味 | 挙動 |
| --- | --- | --- |
| はい | 全て修正 | 数式以外の対象セルを色に関係なく修正する |
| いいえ | 未意思入れ商品のみ修正 | 通常色またはマクロ書込済み色のセルだけ修正する |
| キャンセル | 中止 | 処理を実行しない |

初期選択は `はい` とする。

### 5.6 通常の調整対象判定

通常の販売計画・フェイス陳列数の自動計算対象は、以下の条件で判定する。

```text
本部推奨フェイス1 > 指定下限値
```

この判定では、既存のフェイス陳列数ではなく、本部推奨フェイス1を使用する。店型２陳列数(本部推奨)（本部推奨フェイス2）は判定に使用しない。

本部推奨フェイス1と本部推奨フェイス2がどちらも 0 の対象週は、その週の販売予定がない新商品相当として扱う。この場合、販売計画、フェイス陳列数、既存値の上下限補正のいずれも行わず、値を一切書き換えない。

本部推奨フェイス1が 0 で店型２陳列数(本部推奨)にのみ数値がある対象週は、販売計画・フェイス陳列数の計算や店型2加算を行わず、シート上のフェイス陳列数下限・上限から外れた既存値の補正だけを行う（自動調整・手動調整のみ。HB食品用調整ではこの週を販売予定なしとして扱う）。

HB食品用調整の販売予定なし判定は、本部推奨フェイス1のみで行う（5.4 参照）。

直近3週の販売計画行と実績行がすべて 0 の新商品相当は、過去実績倍率を使えないためSKUブロック全体をスキップし、販売計画、フェイス陳列数、既存値の上下限補正のいずれも行わない。

対象列のすべてが新商品相当の場合は、直近実績に基づく倍率計算も行わず、そのSKUブロック全体をスキップする。

### 5.7 フェイス調整の優先順位

フェイス陳列数の調整は、以下の優先順位で決める。

1. MDシステム取込用のシート上下限値を必ず守る。
2. 目標手持週数を満たすための必要フェイス数を優先する。
3. 本部推奨フェイス1と運用側下限値の大小関係で、下限側の寄せ先を決める。

本部推奨フェイス1が運用側下限値より大きい場合、下限側の寄せ先は運用側下限値とする。

| 条件 | 下限側の寄せ先 |
| --- | --- |
| 本部推奨フェイス1 > 指定下限値 > 既存フェイス陳列数 | 指定下限値 |
| 本部推奨フェイス1 > 既存フェイス陳列数 > 指定下限値 | 指定下限値 |
| 既存フェイス陳列数 > 本部推奨フェイス1 > 指定下限値 | 指定下限値 |

本部推奨フェイス1が運用側下限値より小さい場合、下限側の寄せ先は本部推奨フェイス1とする。

| 条件 | 調整先 |
| --- | --- |
| 指定下限値 > 既存フェイス陳列数 > 本部推奨フェイス1 | 本部推奨値 |
| 既存フェイス陳列数 > 指定下限値 > 本部推奨フェイス1 | 本部推奨値 |
| 指定下限値 > 本部推奨フェイス1 > 既存フェイス陳列数 | 本部推奨値 |

ただし、上表の寄せ先よりも、目標手持週数から見た必要フェイス数が大きい場合は、売れ筋商品のフェイスを意図せず下げないため手持週数側の値を優先する。

例:

```text
指定下限値 = 5
本部推奨フェイス1 = 4
既存フェイス陳列数 = 3
目標手持週数から見た必要フェイス数 = 6
結果 = 6
```

実際に書き込む値は、MDシステム取込エラーを避けるため、シート上のフェイス陳列数下限・フェイス陳列数上限の範囲内に必ず収める。この clamp 条件は全バージョンで必須とする。

### 5.8 販売計画の計算

販売計画は、直近3週の自店実績と本部計画から倍率を計算し、本部計画に掛けて算出する。

算出後の販売計画は、販売計画下限と販売計画上限の範囲内に収める。

### 5.9 フェイス陳列数の計算

フェイス陳列数は、販売計画と目標手持週数から算出する。

算出値は 5.7 の優先順位で決め、最後にシート上のフェイス陳列数下限・フェイス陳列数上限の範囲内に収める。

「店型２陳列数(本部推奨)」（本部推奨フェイス2、`planRow + 13`）に数値が入っている場合は、算出した候補値へその数値をそのまま追加する（半分等の換算は行わない）。空白、数値以外、0 以下の場合は追加しない。追加後の値にも、シート上のフェイス陳列数下限・フェイス陳列数上限の clamp を必ず適用する。

この加算は自動調整・手動調整で行い、HB食品用調整（5.4 参照）では行わない。

### 5.10 意思入れ済みセルの扱い

`全て修正` を選んだ場合、数式セルを除き、色に関係なく対象セルを書き換える。

`未意思入れ商品のみ修正` を選んだ場合、通常色またはマクロ書込済み色のセルのみを書き換える。オレンジ色の意思入れ済みセルは原則として書き換えない。

ただし、MDシステム取込エラーを避けるため、意思入れ済みフェイス陳列数がシート上のフェイス陳列数下限・フェイス陳列数上限の範囲外にある場合だけ範囲内へ補正する。この補正は、本部推奨フェイス1と運用側下限値の大小関係（通常調整対象かどうか）に関係なく、販売予定のある週すべてに適用する。販売予定なしの週と新商品相当の商品は、販売計画・フェイス陳列数・上下限補正のいずれも行わない。

### 5.11 変更記録

値を書き換えた場合、以下を実施する。

- 対象セルをマクロ書込済み色にする。
- 対象行の B 列に `V` を入れる。
- `change_flag_sheet` の対象セル位置にも `V` を入れる。

`change_flag_sheet` は必須シートとする。存在しない場合、変更フラグを記録できないため処理を中止する。

### 5.12 調整後ブック保存

調整後ブックは、条件とあなたの名前が分かる名前で `SaveAs` 保存する。

保存名は以下の順序で統一する。

- 日付
- 対象商品の名称情報4文字
- 入力されたあなたの名前
- 自動調整・手動調整・HB食品用調整の目標手持週数
- 運用側最下限値
- 未意思入れ商品のみ修正の場合だけ、条件名の後ろに `_未意思入のみ`
- 自動調整の場合だけ、最後に `_2週間分`

基本形は `mmdd商品名4文字入力名_X.X週_下限値N` とする。
未意思入れ商品のみ修正の場合は `mmdd商品名4文字入力名_X.X週_下限値N_未意思入のみ` とする。
例: `0520コミュニ山田_1.9週_下限値5_未意思入のみ.xlsx`

自動調整の場合は、上記の末尾に `_2週間分` を付ける。
例: `0520コミュニ山田_2.0週_下限値5_2週間分.xlsx`
未意思入れ商品のみ修正を選んだ自動調整では、`0520コミュニ山田_2.0週_下限値5_未意思入のみ_2週間分.xlsx` とする。

HB食品用調整の保存名は手動調整と同形式とし、専用の接尾辞は付けない。

同じ保存名のファイルが既にある場合は、旧ファイルを削除してから保存する。`_2`、`_3` などの連番は付けない。

対象商品の名称情報は、最初に見つかった商品ブロックの D 列から取得する。D 列が `067:コミュニケーショ` のような文字列の場合、コロンの右側を取り出し、先頭4文字だけを保存名へ使う。例: `コミュニ`

あなたの名前の入力ダイアログの既定値は、`list` シートの C6 以下にあるデパ名から作る「コロン右側の先頭3文字＋担当」とする。例: `110:ファニチャー` → `ファニ担当`。C6 から下へ見て、コロン（半角・全角）を含む最初のセルをデパ名として使う。コロンを含むセルが見つからない場合は Excel のユーザー名を既定値とする。既定値は編集可能とし、空欄のままでは先へ進めない。

調整後ブックの保存先は、操作PCの Downloads フォルダとする。元ブックと同じフォルダには保存しない。ブラウザからダウンロードして直接開いたブックは、元フォルダがキャッシュ等の分かりにくい場所を指し、MDシステムへの取込時に調整後ブックを見失うためである。

Downloads フォルダは、ブラウザの保存先と一致させるため Windows の既知フォルダ設定（`shell:Downloads`）を最優先で解決する。Downloads を既定以外の場所へ移動しているPCでも移動先へ保存する。

Downloads フォルダが見つからないPCに限り、フォルダ選択ダイアログで保存先を指定する。

調整後ブックの保存は必須とする。フォルダ選択ダイアログになった場合、キャンセル時は選び直しを促し、最終的に保存先が選ばれなければ完了扱いにしない。

保存成功後、元ファイルと新ファイルのパスが異なる場合は元ファイル削除を試みる。ただし削除するのは、元ファイルが Downloads・一時フォルダ（TEMP/TMP）・ブラウザキャッシュのいずれかの配下にある場合だけとする。共有フォルダ（UNC）やそれ以外の場所にある原本は削除せず保持し、完了メッセージに保持した旨を表示する。

完了 MsgBox には、以下の項目を簡潔に表示する。数値修正後のバックアップ情報は表示しない。

- 販売計画修正数
- フェイス陳列修正数
- 修正対象商品
- 手持週数
- 運用側最下限値
- 保存名
- 元ファイル削除結果
- AM維持フラグ退避結果

修正対象商品には、修正範囲選択で選んだ `全て修正` または `未意思入れ商品のみ修正` を表示する。

### 5.13 列単位脱色

列単位脱色では、操作者が開始列と終了列を指定し、その列範囲内の販売計画行・フェイス陳列数行を脱色対象とする。指定できる列は `Z` 列から `AK` 列までとする。

処理時は以下を行う。

- 販売計画行は、販売計画用の基本色へ戻す。
- フェイス陳列数行は、フェイス陳列数用の基本色へ戻す。
- 対象セルの `change_flag_sheet` フラグを消す。
- `change_bf_data` が存在する場合は、同じセル位置の色も基本色へ戻す。
- 対象行に変更フラグが残っていない場合は、`list` シート B 列の変更印を消す。
- 再エクスポート後に在庫管理ブック側の起動マクロで再着色されないよう、対象列から15列右にある着色フラグも消す。

値そのものは変更しない。

### 5.14 局所脱色

局所脱色では、`list` シート上で操作者が選択しているセルだけを脱色対象とする。脱色できる列は `Z` 列から `AK` 列までとする。

処理時は以下を行う。

- 販売計画行の選択セルは、販売計画用の基本色へ戻す。
- フェイス陳列数行の選択セルは、フェイス陳列数用の基本色へ戻す。
- 対象セルの `change_flag_sheet` フラグを消す。
- `change_bf_data` が存在する場合は、同じセル位置の色も基本色へ戻す。
- 対象行に変更フラグが残っていない場合は、`list` シート B 列の変更印を消す。
- 再エクスポート後に在庫管理ブック側の起動マクロで再着色されないよう、対象列から15列右にある着色フラグも消す。
- 販売計画行・フェイス陳列数行以外の選択セルは値も色も変更しない。

値そのものは変更しない。

### 5.15 AM列フェイス維持フラグ退避

AM列のフェイス陳列数行にある維持フラグ `2` は、MDシステムへ渡せない値として扱う。

AM維持フラグ退避は、単独の実行入口を持たず、自動調整・手動調整・HB食品用調整の実行時に SKU ブロック処理の前処理として既定で行う。

処理時は以下を行う。

- 対象商品の JAN を履歴ブックへ保存する。
- 履歴ブック内に同じ JAN がすでにある場合、同じ JAN は追記しない。
- 履歴ブック内に既存の重複 JAN 行がある場合、保存時に重複行を削除する。
- 履歴保存に成功した後で、対象セルを空白化する。
- 消去したセルには変更印を残す。
- 履歴ブックの保存先は、操作PCの Downloads フォルダとする。調整後ブック（5.12 参照）と同じ方法で解決し、Downloads が見つからないPCに限りフォルダ選択ダイアログで指定する。前回保存先の再利用は行わない。
- 対象の `2` が1件もない場合は、履歴ブックを作成せずそのまま調整処理へ進む。
- 履歴保存先が確定しなかった場合は、値を一切書き換えずに調整処理全体を中止する。
- 退避結果は、調整完了の MsgBox 内に表示する。

### 5.16 選択範囲PDF出力

選択中のセル範囲を、A4片面（A4用紙1ページ）に収めた PDF ファイルとして保存できること。

処理時は以下を行う。

- セル範囲が選択されていない場合、または離れた複数範囲が選択されている場合は処理を中止する。
- 保存先とファイル名はダイアログで指定する。初期位置は操作PCの Downloads フォルダとする。
- 用紙サイズは A4 とし、選択範囲が1ページに収まるよう縮小する。用紙の向きは対象シートの印刷設定に従う。
- 出力後、保存した PDF を開く。
- 対象シートの印刷設定は、実行前の状態へ戻す。

### 5.17 エラー報告

エラー発生時は、処理名、段階、対象ブック、対象シート、販売計画行、フェイス行、列、エラー番号、エラー内容を表示および記録する。

記録先は以下とする。

- 画面の MsgBox
- VBE の Immediate ウィンドウ
- 対象ブック内の `debug_log` シート

## 6. 非機能要件

### 6.1 保守性

処理は責務別のモジュールに分割し、計算、保存、保存先選択、操作者確認、設定、AM退避、エラー報告を分離する。

調整後ブック・履歴ブックの保存先決定（いずれも Downloads 固定）は `modStorageFolders` に集約する。

あなたの名前入力（デパ名からの既定値生成を含む）や修正範囲確認などの文言は `modPrompts` に集約する。

ファイル名生成、履歴ブック保存、調整後保存は `modFileSave` に集約する。

### 6.2 文字コード

VBE 取り込み時の日本語文字化けを避けるため、`src/vba/mod*.bas` は Shift-JIS（CP932）+ CRLF で保存する。

### 6.3 安全性

大量更新中は Excel の画面更新、イベント、再計算を一時停止し、処理終了時またはエラー時に元へ戻す。

運用中の在庫自動調整・在庫手動調整では、数値修正後の元ファイルバックアップを自動作成しない。設計段階や運用前段階の作業では、作業者が必要に応じて別途バックアップを取得する。

## 7. 現時点の制約

- Excel 標準 `MsgBox` ではボタン表記を任意文言へ変更できないため、本文内で `はい：全て修正` のように意味を明示する。
- 自動調整の対象列は `Z` 列および `AA` 列である。
- 手動調整・HB食品用調整の対象列は現在 `Z` 列のみである。
- 脱色可能列は `Z` 列から `AK` 列までである。
- 本部推奨フェイスが小数になる場合、フェイス陳列数として扱うため切り上げ整数にする。
- フェイス陳列数上限が本部推奨値より低い場合、本部推奨値ではなくフェイス陳列数上限までの戻しになる。
- 店型２陳列数(本部推奨)の追加分を含む値も、フェイス陳列数上限を超える場合は上限までの書き込みになる。
- 選択範囲PDF出力は、離れた複数範囲の選択に対応しない。
