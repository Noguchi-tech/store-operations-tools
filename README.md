# 在庫自動調整マクロ

無印良品 イオンタウン菰野向けの Excel アドインです。
対象ブックの `list` シートに対して、販売計画とフェイス陳列数を調整し、MDシステムへ取り込める調整後ブックとして保存します。

## ファイル構成

- `src.xlsm`: 編集元のマクロ有効ブック
- `releases/`: 配布用アドインの配置先
- `vba_modules/mod*.bas`: アドインへ取り込む標準モジュールのエクスポート
- `customUI14.xml`: リボン定義
- `customUI14.xml.rels`: リボン関連ファイル
- `docs/要件定義書_在庫自動調整マクロ.md`: 要件定義
- `docs/設計書_在庫自動調整マクロ.md`: 設計書

## 保存名ルール

自動調整・手動調整で保存する調整後ブック名は、以下に統一します。

```text
mmdd商品名4文字入力名_X.X週_下限値N
```

未意思入れ商品のみ修正の場合だけ、条件名の後ろに `_未意思入のみ` を付けます。

```text
mmdd商品名4文字入力名_X.X週_下限値N_未意思入のみ
```

例: `0520コミュニ山田_1.9週_下限値5_未意思入のみ.xlsx`

自動調整で作成したファイルは、未意思入れ商品のみ修正の有無にかかわらず、最後に `_2週間分` を付けます。

```text
mmdd商品名4文字入力名_2.0週_下限値5_2週間分
mmdd商品名4文字入力名_2.0週_下限値5_未意思入のみ_2週間分
```

同じ保存名のファイルが既にある場合は、旧ファイルを削除してから保存します。`_2`、`_3` などの連番は付けません。

商品名4文字は、最初に見つかった商品ブロックの D 列から取得します。D 列に `:` または `：` がある場合は右側を取り出し、先頭4文字だけを使います。

## 更新時の注意

`vba_modules` 配下の `.bas` は、VBE へ安全に取り込めるよう CP932 で管理します。
コードを変更した場合は、`src.xlsm` と配布用 `releases/在庫自動調整.xlam` へ同じ変更を取り込み、`DebugCompileSmokeTest` でコンパイル確認してください。
