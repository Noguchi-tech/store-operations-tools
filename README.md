# 在庫自動調整マクロ（個店マクロ）

無印良品 イオンタウン菰野向けの Excel アドインです。
対象ブックの `list` シートに対して、販売計画とフェイス陳列数を一定ルールで調整し、
MDシステムへ取り込める「調整後ブック」を操作PCの **Downloads フォルダ** へ保存します。

## フォルダ構成

```text
store-operations-tools/
├── README.md                       … 最初に読むファイル（このファイル）
├── docs/
│   ├── 開発ガイド.md               … 保守・反映・配布の実務手順
│   ├── 要件定義書_在庫自動調整マクロ.md
│   └── 設計書_在庫自動調整マクロ.md
├── src/                            … 編集対象のソース一式
│   ├── vba/mod*.bas                … VBA 標準モジュール（コードの正本。CP932・CRLF）
│   ├── ribbon/customUI14.xml       … リボン（「便利ツール」タブ）の定義
│   └── src.xlsm                    … 編集元のマクロ有効ブック（.bas 取込先その1）
├── releases/
│   └── 在庫自動調整.xlam           … 配布用アドイン（.bas 取込先その2。利用者へ渡すのはこれ）
└── tools/
    └── sync_vba_to_books.ps1       … .bas とリボンを両ブックへ自動反映・検証するスクリプト
```

**コードの正本は `src/vba/*.bas` です。** `src.xlsm` と `在庫自動調整.xlam` の中の VBA は
このフォルダから取り込んだコピーであり、直接 VBE 上だけで編集して終わりにしないでください。

## 機能の概要

リボンに「便利ツール」タブが追加され、以下を実行できます。

| ボタン | 内容 |
| --- | --- |
| 自動調整 | 手持2.0週・下限5の固定条件で、Z・AA列（2週間分）の販売計画とフェイス陳列数を調整 |
| 手動調整 | 手持週数と下限値を入力して、Z列を調整 |
| 列単位脱色 | 指定した列範囲（Z〜AK）の色と変更フラグを通常状態へ戻す（値は変えない） |
| 局所脱色 | 選択セルだけを脱色（値は変えない） |
| 選択範囲PDF出力 | 選択セル範囲を A4 片面1ページの PDF として保存 |

自動調整・手動調整の実行時は、前処理として AM列の維持フラグ「2」を履歴ブックへ退避してから調整します。

## 調整後ブックの保存ルール

- **保存先は操作PCの Downloads フォルダ固定**です。元ブックと同じフォルダには保存しません。
  ブラウザからダウンロードして直接開いたブックは元フォルダがキャッシュ等の分かりにくい場所になり、
  MDシステムへの取込時にファイルを見失うためです（Downloads が無いPCのみフォルダ選択ダイアログ）。
- 保存名は以下に統一します。

```text
mmdd商品名4文字入力名_X.X週_下限値N
```

- 未意思入れ商品のみ修正の場合だけ `_未意思入のみ` を付けます。
- 自動調整では最後に `_2週間分` を付けます。
- 同名ファイルが既にある場合は旧ファイルを削除してから保存します（`_2` などの連番は付けません）。
- 商品名4文字は最初の商品ブロックの D 列「コード:名称」のコロン右側の先頭4文字です。
- 「入力名」（あなたの名前）の既定値は、`list` シート C6 以下のデパ名から自動生成される
  「コロン右側の先頭3文字＋担当」です。例: `110:ファニチャー` → `ファニ担当`

例: `0520コミュニファニ担当_2.0週_下限値5_2週間分.xlsx`

## コードを変更したら（日常の3ステップ）

```powershell
# 1. src/vba/*.bas を編集する（文字コード CP932・改行 CRLF を維持。詳細は開発ガイド）

# 2. 両ブックへ反映して自動検証する（Excel は閉じておく）
pwsh -File tools/sync_vba_to_books.ps1

# 3. コミットして push する
git add -A
git commit -m "変更内容"
git push
```

`sync_vba_to_books.ps1` は、バックアップ → リボン反映 → モジュール入替 →
コンパイル確認 → smoke テスト → 取込結果の diff 検証 まで自動で行います。
失敗した場合はメッセージに従い、[docs/開発ガイド.md](docs/開発ガイド.md) のトラブルシューティングを参照してください。
